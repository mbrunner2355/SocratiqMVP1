## üìù Typography

### Font Family
**Primary Font**: Roboto, Open Sans, Helvetica (Sans-serif priority)
- Highly legible across all screen sizes
- Clean, modern appearance
- Excellent cross-platform support

### Font Weights & Hierarchy
- **Bold (700)**: Headings, key performance metrics, high-level indices
- **Semi-Bold (600)**: Section headers, important labels
- **Regular (400)**: Body text, detailed metric descriptions
- **Light (300)**: Supplementary text, captions

### Font Sizes (Responsive)

#### Desktop
- **H1 Headings**: 20pt (1.25rem)
- **H2 Headings**: 18pt (1.125rem)
- **H3 Headings**: 16pt (1rem)
- **Body Text**: 14pt (0.875rem)
- **Small Text**: 12pt (0.75rem)

#### Tablet/Mobile
- **H1 Headings**: 18pt (1.125rem)
- **H2 Headings**: 16pt (1rem)  
- **H3 Headings**: 14pt (0.875rem)
- **Body Text**: 13pt (0.8125rem)
- **Small Text**: 11pt (0.6875rem)

### Typography Principles
- Consistent hierarchy across all device types
- Key metrics use larger fonts on mobile for visibility
- Maintain readability without screen overcrowding

## ‚ôø Accessibility

### Compliance Standards
- **WCAG 2.1 AA** compliance minimum
- **Color contrast ratio**: 4.5:1 for normal text, 3:1 for large text
- **Color-blind friendly**: Never rely solely on color for information
- **Keyboard navigation**: Full platform accessibility via keyboard
- **Screen reader support**: Semantic HTML and ARIA labels

### Implementation
- Icons/labels supplement color-coding for data visualization
- Focus indicators clearly visible for all interactive elements
- Alt text for all meaningful images and charts
- Proper heading structure (H1-H6) for screen readers

## üíª Technical Implementation

### CSS Framework Approach
- **Utility-first**: Tailwind CSS compatible classes
- **Component-based**: Reusable component library
- **CSS Custom Properties**: For dynamic theming
- **Responsive utilities**: Mobile-first breakpoint system

### Replit.com + Tab9 Considerations
- Optimize for web-based development environment
- Ensure components work across different browsers
- Consider performance for cloud-based development
- Maintain compatibility with various deployment targets

### Performance Guidelines
- **Images**: WebP format with fallbacks, lazy loading
- **Fonts**: Subset fonts, preload critical fonts
- **CSS**: Critical CSS inlined, progressive enhancement
- **JavaScript**: Module bundling, code splitting for large apps
